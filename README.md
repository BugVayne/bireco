# ClearMetrics — сайт BI-консалтинга

Статический сайт (HTML/CSS/JS) + бесплатный бэкенд на Google Apps Script:
заявки с формы и новости хранятся в Google Таблице, новости добавляются через
страницу-админку `admin.html`. Для SEO небольшой скрипт `build.py` собирает
из исходников готовые EN/RU-страницы и отдельную страницу на каждую новость.

## Структура

```
index.html            — главная (Hero, Insights, Services, Industries, Careers, About Us)
news.html             — все новости развёрнуто
admin.html            — админка новостей (вход по паролю)
css/styles.css        — стили
js/config.js          — НАСТРОЙКИ (URL бэкенда, ключ reCAPTCHA, контакты)
js/i18n.js            — переводы EN/RU и переключатель языка
js/services-data.js   — тексты сервисов (EN/RU) + демо-новости
js/news-api.js        — загрузка новостей
js/main.js            — логика главной (карусель, форма, видео)
js/news-page.js       — логика страницы новостей
js/admin.js           — логика админки
js/ui.js              — полоса прогресса прокрутки и кнопка «наверх»
apps-script/Code.gs   — код бэкенда для Google Apps Script
build.py              — генератор статических EN/RU и новостных страниц (SEO)
requirements.txt      — зависимости build.py
```

Пока бэкенд не настроен, сайт показывает демо-новости из `js/services-data.js`,
а форма выводит подсказку вместо отправки — локально всё можно посмотреть сразу.

> `index.html` / `news.html` остаются полноценными рабочими страницами — их можно
> открыть напрямую. `build.py` использует их как шаблоны и кладёт собранный сайт
> в папку `dist/` (она в `.gitignore` — её создаёт деплой, коммитить не нужно).

## Шаг 1. Google Таблица + Apps Script (15 минут)

1. Создайте новую Google Таблицу: https://sheets.new
   Листы `News` и `Leads` создадутся автоматически при первом обращении.
2. В таблице: **Расширения → Apps Script**.
3. Удалите содержимое `Code.gs` и вставьте код из `apps-script/Code.gs` этого проекта.
4. Слева **⚙ Project Settings → Script Properties → Add script property**:
   - `ADMIN_TOKEN` — придумайте длинный пароль для админки (например, 20+ символов);
   - `RECAPTCHA_SECRET` — секретный ключ reCAPTCHA (см. Шаг 2; можно добавить позже);
   - `NOTIFY_EMAIL` — (необязательно) почта для уведомлений о новых заявках.
5. **Deploy → New deployment → ⚙ Web app**:
   - Execute as: **Me**
   - Who has access: **Anyone**
   - Нажмите Deploy, разрешите доступ, скопируйте **URL веб-приложения** (`https://script.google.com/macros/s/…/exec`).
6. Вставьте этот URL в `js/config.js` → `APPS_SCRIPT_URL`.

> Важно: после любого изменения кода в Apps Script делайте
> **Deploy → Manage deployments → ✏ → New version**, иначе изменения не применятся.

## Шаг 2. reCAPTCHA (5 минут)

1. Откройте https://www.google.com/recaptcha/admin/create
2. Тип: **reCAPTCHA v2 → "I'm not a robot" Checkbox**.
3. Домены: добавьте домен сайта (и `localhost` для тестирования).
4. **Ключ сайта (site key)** → в `js/config.js` → `RECAPTCHA_SITE_KEY`.
5. **Секретный ключ (secret key)** → в Script Properties → `RECAPTCHA_SECRET` (Шаг 1.4).

Без ключей форма работает, но без капчи (остаётся honeypot-защита от простых ботов).

## Шаг 3. Публикация сайта (настроено)

Сайт публикуется автоматически через GitHub Pages:

- Репозиторий: https://github.com/BugVayne/bireco
- Workflow `.github/workflows/deploy.yml` запускает `build.py` (генерирует EN/RU
  и новостные страницы), затем выкладывает папку `dist/` на
  **https://bugvayne.github.io/bireco/**.
- Триггеры деплоя: каждый `git push` в `main`; **каждую ночь в 03:00 UTC**
  (чтобы новости из админки превратились в статические страницы); вручную —
  вкладка **Actions → Deploy to GitHub Pages → Run workflow**.
- Статус деплоя: вкладка **Actions** в репозитории.

Откат к предыдущей версии: `git revert <commit>` + `git push` (или Re-run старого
workflow в Actions).

При переезде на собственный домен поменяйте константу `SITE` в начале `build.py`
и URL в `robots.txt` — `sitemap.xml`, `canonical` и hreflang пересоберутся сами.
Не забудьте добавить домен в настройки reCAPTCHA (Шаг 2.3).

## SEO: как устроены EN/RU и новости

Переключение языка и новости работают в браузере, поэтому без пререндера
поисковик увидел бы только английскую «оболочку» без новостей. `build.py` решает
это, заранее собирая обычные HTML-страницы:

```
/            и  /ru/             — главная на двух языках
/news.html   и  /ru/news.html    — список новостей
/news/<id>.html и /ru/news/<id>.html — отдельная страница каждой новости
sitemap.xml                      — все URL с hreflang-связками EN↔RU
```

У каждой страницы свои `<title>`, `<meta description>`, Open Graph, `canonical`
и `hreflang` — Google индексирует обе языковые версии и каждую новость отдельно.
Тексты берутся из тех же `js/i18n.js`, `js/services-data.js` и из Google Таблицы —
**дублировать контент не нужно**, правки в одном месте попадают и в собранные страницы.

Запустить сборку локально (по желанию, для проверки):

```
pip install -r requirements.txt
python build.py        # результат — в папке dist/
```

Если SEO не важен (клиенты приходят по прямой ссылке/рекомендациям) — можно ничего
не запускать руками: деплой делает это сам. Файл `build.py` самодостаточен и
переезжает с проектом на любой хостинг.

## Как добавлять новости

1. Откройте `https://ваш-сайт/admin.html`.
2. Введите пароль (`ADMIN_TOKEN` из Шага 1.4).
3. Заполните форму (дата, заголовки и тексты на EN и RU, ссылка на картинку) → **Сохранить**.
4. Новость сразу появится в ленте на главной и на странице «Все новости».

Альтернатива: новости можно править прямо в Google Таблице на листе `News`
(колонки: `id, date, title_en, title_ru, summary_en, summary_ru, body_en, body_ru, image`).
Дату указывайте в формате `ГГГГ-ММ-ДД`.

Картинки для новостей: загрузите файл на любой хостинг изображений
(или в папку сайта) и вставьте прямую ссылку в поле `image`.

## Заявки с формы

Каждая отправка формы попадает в лист `Leads`
(время, имя, email, телефон, сообщение, сервис, язык, профиль, ссылка на резюме).
Если задан `NOTIFY_EMAIL`, на него приходит письмо о каждой заявке.

Защита от спама (несколько уровней):
- **reCAPTCHA v2** — проверяется на сервере;
- **honeypot** — скрытое поле, которое заполняют только боты;
- **лимит частоты** — не больше 5 заявок в час с одного email и не больше 50 в час
  суммарно (значения `LEAD_LIMIT_PER_EMAIL` / `LEAD_LIMIT_TOTAL` в начале `Code.gs`).

### Отклики на вакансии (загрузка резюме)

Кнопка «Отправить резюме» в блоке Careers открывает ту же форму с двумя
дополнительными полями: **ссылка на профиль** (LinkedIn / hh и т.п.) и
**прикрепить CV** (файл `.pdf`, `.doc`, `.docx`, до 5 МБ).

Файл резюме сохраняется на ваш **Google Drive** (в папку `Website CV uploads`,
создаётся автоматически), а прямая ссылка на него попадает в колонку `cv` листа
`Leads` и в письмо-уведомление. Файл остаётся приватным; если задан
`NOTIFY_EMAIL`, этому адресу выдаётся доступ на просмотр.

Что учесть при настройке:
- При первом запуске после вставки нового `Code.gs` Apps Script **попросит
  дополнительное разрешение на доступ к Google Drive** — это нормально, подтвердите.
- Чтобы складывать резюме в конкретную папку, добавьте в Script Properties
  свойство `CV_FOLDER_ID` со значением ID папки (часть ссылки `…/folders/<ID>`);
  без него используется папка `Website CV uploads` в корне Drive.
- Если лист `Leads` уже существовал со старыми колонками, новые `profile` и `cv`
  допишутся в конец строки; при желании добавьте их заголовки в первую строку вручную.

## Безопасность админки

После входа Apps Script выдаёт **сессию на 2 часа** (константа `SESSION_TTL_SEC`),
которая продлевается при каждом действии. Пароль `ADMIN_TOKEN` передаётся только
при входе, дальше используется одноразовый идентификатор сессии. По истечении
срока админка просит войти заново. «Выйти» завершает сессию вручную.

Записи в таблицу (заявки и операции с новостями) защищены `LockService`, чтобы
одновременные обращения не перезаписали друг друга.

## Что поменять под себя

- **Название и логотип**: «ClearMetrics» в `index.html`, `news.html`, `admin.html` (поиск по тексту), логотип — блок `.logo`.
- **Контакты в футере**: `js/config.js` → `CONTACT_EMAIL`, `LINKEDIN_URL`.
- **Видео для "Watch overview"**: `js/config.js` → `OVERVIEW_VIDEO_URL`
  (embed-ссылка YouTube вида `https://www.youtube.com/embed/XXXX`).
  Пока пусто — кнопка прокручивает к сервисам.
- **Тексты сервисов**: `js/services-data.js` (EN и RU).
- **Тексты разделов** (hero, Industries, Careers, About): `js/i18n.js`.
- **Цвета**: переменные в начале `css/styles.css` (`--navy`, `--blue`, …).
